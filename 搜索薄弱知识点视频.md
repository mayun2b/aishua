## 薄弱知识点视频搜索功能实现思路

### 一、功能整体设计

这个功能将作为现有学习分析模块的**增强功能**，在用户完成学习分析后，可以为识别出的薄弱知识点自动搜索相关讲解视频。

**用户流程：**
1. 用户在学情分析页面完成分析，查看薄弱知识点
2. 点击某个薄弱知识点的「找讲解视频」按钮
3. 系统后台自动搜索相关视频
4. 前端展示搜索结果（视频标题、来源、链接、预览图等）

---

### 二、技术方案选择

#### 2.1 视频搜索方式（推荐方案优先级）

**方案一：使用成熟的搜索 API（推荐优先）**
- **Bing Video Search API**：Microsoft 提供，质量较高，有免费额度
- **Google Custom Search JSON API**：需要配置自定义搜索引擎，可限制只搜视频
- **SerpAPI**：封装了 Google/Bing 搜索，使用简单但需要付费

**方案二：利用现有的 AI 能力**
- 调用 Qwen/通义千问等大模型，让它帮你「思考」搜索关键词，并给出推荐的视频平台（B站、YouTube等）
- 再根据大模型返回的建议，构造直接访问视频平台的搜索链接

**方案三：直接对接视频平台 API**
- Bilibili API（有公开的搜索接口）
- YouTube Data API v3（需要申请密钥）

**推荐组合：** 先用方案二的大模型优化搜索词，再用方案一的搜索 API 获取结果，或者直接构造到 B站/优酷的搜索结果页链接。

### 三、后端设计思路

#### 3.1 新增模块
在现有 `modules/ai/` 下新增 `video` 子模块（或复用现有结构）：

**核心组件：**
1. **VideoSearchController**：提供 API 接口

#### 3.2 接口设计
```
POST /api/ai/video-search
请求参数：
- knowledgePointName: 知识点名称
- subjectName: 学科名称（可选，用于优化搜索）
- grade: 年级（可选，用于优化搜索）
- limit: 返回结果数量（默认 5）

响应结构：
- success: boolean
- videos: 视频列表
  - title: 视频标题
  - url: 视频链接
  - source: 来源平台（如 "bilibili", "youku"）
  - thumbnail: 预览图链接（可选）
  - duration: 时长（可选）
  - description: 简介（可选）
```

#### 3.3 配置文件扩展
在 `application.yml` 中新增视频搜索相关配置：
```yaml
ai:
  video-search:
    enabled: true
    default-limit: 5
    bing:
      api-key: ${BING_VIDEO_API_KEY:}
      endpoint: https://api.bing.microsoft.com/v7.0/videos/search
    # 或者 bilibili 的配置
    bilibili:
      search-url: https://search.bilibili.com/all
```

#### 3.4 搜索策略流程（推荐）

**「轻量级」搜索流程（无需外部 API 费用）：**
1. 接收用户的知识点名称
2. 调用现有的 `QwenChatClient`，让大模型帮你**优化搜索词**
   - 示例 Prompt：`请为知识点「{知识点}」生成 3 个适合在 B站 搜索教学视频的关键词，返回 JSON 数组格式`
3. 构造 B站/优酷等平台的搜索链接：`https://search.bilibili.com/all?keyword={编码后的关键词}`
4. 直接返回这些「智能搜索链接」给前端，用户点击后跳转到对应平台的搜索结果页

**「重量级」搜索流程（需要 API）：**
1. 接收知识点
2. 调用大模型优化搜索词
3. 调用 Bing Video Search API 或 Bilibili API 获取真实视频列表
4. 过滤、排序、去重后返回

## 视频站内播放设计方案

### 一、核心结论

| 平台 | 能否 iframe 嵌入播放 | 状态 |
|------|---------------------|------|
| **B站** | **可以** | 官方支持，用 `player.bilibili.com/player.html?bvid=xxx` |
| **优酷** | **不可以** | 2023年底已关闭第三方嵌入，iframe 会显示"视频不可用" |
| **腾讯视频** | **可以** | 支持 `v.qq.com/iframe/player.html?vid=xxx` |
| **YouTube** | **可以** | 官方支持 iframe embed |

**结论：只保留 B站 作为站内播放源，去掉优酷。** 如果后续需要，可以加腾讯视频。

### 三、后端设计

#### 3.1 需要获取的数据

要实现站内播放，后端必须返回**真实的视频信息**（而不是搜索链接），至少包含：

- `bvid`：B站视频唯一标识（用于构造 iframe 播放地址）
- `title`：视频标题
- `pic`：视频封面图 URL

- `owner`：UP主名称
- `play`：播放量

#### 3.2 获取视频列表的方式（推荐优先级）

**方案 A：调用 B站搜索 API（推荐）**

```

返回结构化的视频列表数据，包含 bvid、标题、封面等。

4. 返回视频列表

**方案 C：后端直接用 RestTemplate 调用 B站 API**

最简单，直接在 `VideoSearchService` 中用 `RestTemplate` 请求 B站搜索接口。

#### 3.3 推荐方案：C（后端直接调用 B站搜索 API）

### 4. LearningAnalysisService（学情分析服务）如何调用 Dify

项目中没有直接叫 `LearningAnalysisService` 的类，实际负责学情分析与 Dify 交互的是 **`DifyTestService`** 和 **`LearningAnalysisReportServiceImpl`** 两个类。

#### 调用链路

```

    -> DifyTestService.testDify(DifyTestRequest)
        -> DifyClient.runWorkflow(...)        // 调用 Dify 工作流 API
        -> LearningAnalysisReportService.saveDifyResult(...)  // 解析并持久化结果

#### DifyTestService

**文件路径**: `e:\it\VScode\project\aishua\aishua-master\src\main\java\zysy\iflytek\aishua\modules\dify\service\DifyTestService.java`

**调用 Dify 的核心流程**:

1. 从 `UserContext` 获取当前登录用户的 `userId` 和 `authorization` 令牌
2. 将令牌中的 "Bearer " 前缀去掉，得到纯 token
3. 如果前端未传 `studentId`，默认使用当前登录用户 ID
4. 调用 `difyClient.runWorkflow(query, studentId, userId, plainToken, subject, subjectId, grade, textbookVersion)` 发起 Dify 工作流
5. 将 Dify 返回结果交给 `learningAnalysisReportService.saveDifyResult()` 进行解析和存储
6. 将 Dify 原始响应和持久化后的 `analysisReport` 一起返回

#### LearningAnalysisReportServiceImpl

**文件路径**: `e:\it\VScode\project\aishua\aishua-master\src\main\java\zysy\iflytek\aishua\modules\ai\service\impl\LearningAnalysisReportServiceImpl.java`

`saveDifyResult` 方法负责解析 Dify 工作流返回的 JSON，提取以下信息并入库：

- 从 `data.outputs` 或 `data.structured_output` 中提取 `summary`（总结）、`fullText`（全文）、知识点列表
- 解析 `weakKnowledgePoints` / `weak_knowledge_points` 为 `LearningAnalysisKnowledgePoint` 实体（含 name、module、reason、masteryLevel、correctRate、sampleCount、wrongCount、priority 等字段）
- 解析 `dataQuality`（数据质量是否充足、缺失指标列表）
- 记录 `workflowRunId`、`taskId`、`errorMessage`
- 同一 `workflowRunId` 的报告会更新而非重复插入
- 最终返回 `LearningAnalysisReportVO`

## Dify 工作流优化设计

### 工作流节点设计（4 个节点）

```
节点1：大模型优化搜索词
    ↓
节点2：HTTP请求节点 → 调用B站搜索API（Dify自带HTTP请求能力，从Dify服务器发起，不受你后端IP限制）
    ↓
节点3：代码节点 → 清洗+过滤+格式化
    ↓
节点4：输出
```

### 节点1：大模型优化搜索词

```
你是一个教学视频搜索专家。

知识点：{{knowledge_point}}
学科：{{subject}}
年级：{{grade}}

请生成 3 个适合在B站搜索教学视频的关键词，要求：
1. 精准匹配该知识点的教学讲解
2. 优先匹配高中/初中等对应年级
3. 返回 JSON 数组格式：["关键词1", "关键词2", "关键词3"]
```

### 节点2：HTTP 请求节点（关键！）

**这是解决 B站反爬的关键**：Dify 服务器发起的 HTTP 请求和你后端服务器不同，B站对 Dify 的 IP 没有封禁记录。

```
方法：GET
URL：https://api.bilibili.com/x/web-interface/search/type
参数：
  - search_type: video
  - keyword: {{节点1输出的第一个关键词}}
  - page: 1
  - page_size: 10
请求头：
  - User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
  - Referer: https://www.bilibili.com
```

### 节点3：代码节点（清洗+过滤）

```python
import json

def main(search_result, limit=5):
    """清洗B站搜索结果，过滤不合适的视频"""
    videos = []
    
    # 解析搜索结果
    data = json.loads(search_result)
    results = data.get("data", {}).get("result", [])
    
    for item in results:
        bvid = item.get("bvid", "")
        title = item.get("title", "").replace('<em class="keyword">', '').replace('</em>', '')
        duration_str = item.get("duration", "0:00")
        author = item.get("author", "")
        pic = item.get("pic", "")
        play = item.get("play", 0)
        
        # === 清洗规则 ===
        
        # 1. bvid 格式校验：必须以 BV 开头，长度为12位
        if not bvid or not bvid.startswith("BV") or len(bvid) != 12:
            continue
        
        # 2. 时长过滤：教学视频应在 2分钟~60分钟之间
        duration_parts = duration_str.split(":")
        if len(duration_parts) == 2:
            minutes = int(duration_parts[0])
            seconds = int(duration_parts[1])
            total_seconds = minutes * 60 + seconds
        elif len(duration_parts) == 3:
            hours = int(duration_parts[0])
            minutes = int(duration_parts[1])
            seconds = int(duration_parts[2])
            total_seconds = hours * 3600 + minutes * 60 + seconds
        else:
            continue
        
        # 太短（<2分钟）或太长（>60分钟）的跳过
        if total_seconds < 120 or total_seconds > 3600:
            continue
        
        # 3. 播放量过滤：至少100播放（过滤掉几乎没人看的低质量视频）
        if play < 100:
            continue
        
        # 4. 标题过滤：排除广告、无关内容
        skip_keywords = ["推广", "广告", "恰饭", "抽奖", "福利"]
        if any(kw in title for kw in skip_keywords):
            continue
        
        # 5. 封面图补全协议
        if pic.startswith("//"):
            pic = "https:" + pic
        
        videos.append({
            "bvid": bvid,
            "title": title,
            "pic": pic,
            "duration": duration_str,
            "owner": author,
            "play": play
        })
        
        if len(videos) >= limit:
            break
    
    return {"videos": json.dumps(videos, ensure_ascii=False)}
```

### 节点4：输出

输出变量 `videos`，对应节点3的清洗结果。
