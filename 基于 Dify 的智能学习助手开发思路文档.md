# 智能学习助手模块开发计划

## 1. 模块目标

在现有 `aishua` 学习/刷题项目中集成一个智能学习助手模块，为用户提供考点问答、题型解题思路、错题分析、学习资源推荐和实时网页资料检索能力。

该模块不只是一个聊天窗口，而是要和题目、错题本、考点、学习报告等业务场景结合，成为用户刷题过程中的 AI 学习教练。

## 2. 核心能力

### 2.1 知识问答

用户可以询问某个考点、概念、公式、题型特点，智能体需要给出清晰解释。

示例：

- “函数单调性怎么判断？”
- “数列递推题一般从哪里入手？”
- “这个考点常见题型有哪些？”

### 2.2 解题思路引导

当用户粘贴题目或从题目详情页发起提问时，智能体优先提供解题切入点、关键步骤和易错点，而不是直接给最终答案。

推荐输出结构：

- 题型判断
- 涉及考点
- 入手方向
- 解题步骤
- 易错点
- 同类题练习建议

### 2.3 个性化学习资源推荐

根据用户当前掌握情况、错题记录、薄弱考点和学习阶段，推荐适合的资料。

资源类型包括：

- 视频讲解
- 图文教程
- 专项练习
- 真题训练
- 基础巩固资料

### 2.4 实时网页资料检索

当本地知识库或题库资源不足时，后端通过搜索 API 或网页抓取工具实时获取资料，并进行清洗、摘要、去重和质量评分。

## 3. MVP 范围

第一阶段先实现可用闭环，不做过度复杂的全自动智能体。

MVP 包含：

- AI 对话接口
- 考点问答
- 题目解题思路分析
- 基于用户掌握度的简单资源推荐
- 会话记录保存
- 前端 AI 助手入口
- 基础外部资源搜索能力

暂不包含：

- 复杂多 Agent 协作
- 大规模自动爬虫
- 自动生成完整课程体系
- 长期强化学习式推荐排序

## 4. 推荐模块命名

建议模块命名为：

- `study-agent`
- `ai-tutor`
- `learning-assistant`

在项目中可以统一使用 `AiTutor` 或 `StudyAgent` 作为业务命名。

```text

 ↓

9. Agent 请求流程
一次用户提问的完整流程：

1. 前端提交用户问题和上下文
2. 后端识别用户意图
3. 读取用户学习画像
4. 读取题目、错题或考点上下文
5. 检索本地知识库
6. 判断是否需要网页搜索
7. 整理 prompt
8. 调用大模型
9. 生成结构化回答
10. 保存会话和推荐记录
11. 返回前端展示
10. Prompt 设计
10.1 系统提示词
你是一个面向刷题学习场景的智能学习助手。
你的目标是帮助用户理解考点、掌握题型、形成解题路径。
当用户询问题目时，优先给思路、切入点、关键步骤和易错点。
除非用户明确要求完整答案，否则不要直接跳到最终答案。
推荐资料时，需要结合用户掌握情况、科目、考点和难度。
涉及外部网页资料时，必须给出来源和推荐理由。
如果资料不确定或来源质量一般，需要明确说明。

12. 开发里程碑
第一阶段：基础对话能力
预计周期：3-5 天

任务：

新增 AI 对话接口
接入大模型
实现基础 prompt
前端新增 AI 助手抽屉
支持普通考点问答
验收标准：

用户可以正常提问
AI 可以围绕学习场景回答
会话记录可以保存和查看
第二阶段：题目上下文接入
预计周期：4-6 天

任务：

题目详情页增加“问 AI”
请求中携带题目 ID
后端读取题目内容、选项、答案、解析
AI 输出题型、考点、思路和易错点
验收标准：

用户可针对某道题提问
AI 回答能结合题目内容
支持“只给提示”和“完整解析”两种模式
第三阶段：用户掌握度接入
预计周期：5-7 天

任务：

接入用户错题和练习数据
计算薄弱考点
回答中结合用户当前水平
错题本增加“AI 分析错因”
验收标准：

AI 能指出用户薄弱点
推荐内容能区分基础薄弱和进阶提升
错题分析能给出复习建议
第四阶段：资源推荐能力
预计周期：5-8 天

任务：

新增学习资源表
实现资源推荐服务
支持视频、文章、题目资源推荐
前端展示资源卡片
记录用户点击、收藏、忽略行为

AI 能推荐资料并说明理由

用户行为可以被记录

预计周期：7-10 天

任务：

接入搜索 API
实现网页内容抽取
实现资源去重和质量评分

14. 风险与控制
14.1 回答不准确
控制方式：

优先使用本地题库和知识库
重要回答带来源
对不确定内容明确提示
后续加入用户反馈机制
14.2 网页资源质量低
控制方式：

设置来源白名单
对内容进行质量评分
过滤广告页和采集站
展示推荐理由
14.3 大模型成本过高
控制方式：

缓存相似问题回答
缓存网页搜索结果
对长题目做摘要后再请求
区分普通模型和高质量模型使用场景
14.4 用户过度依赖答案
控制方式：

默认给思路，不直接给答案
支持“只给提示”模式
鼓励用户先尝试
回答中加入同类题练习建议
15. 后续迭代方向

16. 推荐开发优先级
建议按以下顺序开发：

题目详情页 AI 解题思路
 → 考点问答
 → 错题本错因分析
 → 用户掌握度个性化推荐
 → 学习资源推荐
 → 实时网页检索
第一版的目标是先让 AI 真正嵌入刷题流程，而不是单独做一个普通聊天机器人。
```
