# 教师端题目图片 MinIO 上传优化设计

## 背景

教师端题库管理的题目新增和编辑表单目前要求手动填写图片地址。项目已经具备通用 MinIO 文件上传能力，前端有 `fileApi.upload(file)`，后端有 `/api/files/upload`，上传成功后返回 `objectName`。

本次优化只改善教师录入体验：题目图片通过文件选择上传到 MinIO，题目数据继续保存 MinIO 对象名到现有 `imageUrls` 字段。

## 目标

- 在教师端题目新增和编辑弹窗中提供图片上传入口。
- 上传成功后保存 MinIO 对象名，而不是要求教师手填 URL。
- 编辑已有题目时能展示已保存的对象名列表，并支持移除。
- 保持现有数据库字段和题目新增、更新接口不变。

## 非目标

- 不调整 `question.image_urls` 的数据库结构。
- 不新增题目专用上传后端接口。
- 不在本次改动中重做学生端题目图片展示。
- 不自动删除从题目中移除的 MinIO 对象，避免误删仍被其他题目引用的文件。

## 设计

### 前端组件边界

改动集中在 `aishua-web/src/modules/admin/views/AdminQuestionManagementView.vue`。

题目表单中的“图片URL”文本输入替换为图片上传区域：

- 使用 `<input type="file" accept="image/*" multiple>` 选择图片。
- 每个文件调用现有 `fileApi.upload(file)`。
- 从响应中读取 `objectName`。
- 将对象名追加到表单内部的图片对象名列表。
- 页面展示已上传对象名，并提供“移除”按钮。

### 数据流

编辑已有题目时：

1. 读取 `question.imageUrls`。
2. 按英文逗号切分。
3. 去除空白和空项。
4. 写入表单图片对象名列表。

上传新图片时：

1. 教师选择一张或多张图片。
2. 前端逐个上传到 `/files/upload`。
3. 上传成功后将返回的 `objectName` 加入列表。
4. 上传失败时显示 toast，不清空已成功上传的图片。

提交题目时：

1. 将图片对象名列表去重。
2. 用英文逗号拼接为字符串。
3. 写回现有 `imageUrls` 字段提交给 `questionApi.create` 或 `questionApi.update`。

### 状态与交互

- 上传过程中禁用上传按钮或展示“上传中”状态，防止重复点击。
- 提交题目时如果仍在上传，阻止提交并提示“图片上传中，请稍后提交”。
- 重置表单时恢复到打开弹窗时的图片对象名列表。
- 移除图片只影响当前题目表单，不调用 MinIO 删除接口。

### 错误处理

- 文件上传失败：显示接口错误消息或默认“图片上传失败”。
- 上传接口没有返回 `objectName`：按失败处理并提示。
- 非图片文件：依赖 `accept="image/*"` 做第一层限制；如浏览器仍传入非图片，前端用 `file.type.startsWith('image/')` 再拦截。

## 测试

前端至少验证这些行为：

- 已有 `imageUrls` 字符串可以解析成图片对象名列表。
- 提交 payload 时图片对象名列表会拼成逗号字符串。
- 上传成功后对象名会加入列表。
- 上传失败不会清空已有图片对象名。
- 上传中不能提交题目。

如果现有项目没有前端单元测试基础，至少运行 `npm run lint` 和 `npm run build`，并用浏览器手动验证题目编辑上传流程。
