# 题目图片标注与记录回看设计

## 背景

题目图片已经支持通过 MinIO 对象名保存，并且作答页可以展示题目配图。当前问题有两个：

- 作答页图片展示区域偏大，图片尺寸不够紧凑，影响答题布局。
- 数学题等场景需要在题图上画辅助线，且这些作图痕迹需要随答案保存或提交，后续查看练习/考试记录时仍能看到。

本设计聚焦学生端作答、保存、提交和记录回看链路。教师端图片上传流程保持现有 MinIO 对象名方案。

## 目标

- 优化作答页题图展示尺寸，减少空白，支持清晰查看。
- 允许学生在题图上绘制辅助线。
- 作图记录随练习草稿、练习提交、考试提交保存。
- 查看练习记录和考试记录时，可以看到原题图片及当时保存的辅助线。
- 不把图片二进制存进数据库；数据库只保存 MinIO 对象名。
- 自动判题只使用纯答案内容，不受作图标记影响。

## 非目标

- 不实现复杂绘图软件能力，例如图层管理、文字标注、几何吸附、撤销到每一笔的历史回放。
- 不修改原题图片内容，不覆盖教师上传的图片。
- 不要求旧记录补全标注数据；旧记录没有标注时只展示原题图片。
- 不在本次设计中引入新的文件删除回收流程。

## 方案选择

### 推荐方案：透明标注层上传 MinIO，答案中保存对象名标记

学生在图片上绘制辅助线时，前端在原图上叠加一层透明 canvas。保存或提交前，把透明 canvas 导出为 PNG 上传到 MinIO，拿到 `annotationObjectName`。答案正文中追加结构化标记，记录原图对象名和标注层对象名。

示例：

```text
用户答案内容
[[image-annotations:[{"imageObjectName":"questions/q043.png","annotationObjectName":"answers/annotations/2026-06-02/xxx.png"}]]]
```

记录回看时，前端解析该标记，下载原题图片和透明标注层，将标注层覆盖到原图上展示。

优点：

- 不新增数据库字段，使用现有 `userAnswer` 文本字段即可保存。
- 原题图片不重复上传，存储成本低。
- 标注层可以和原图分离，记录页能准确复现。
- 对练习和考试可以共用同一套前端解析与上传逻辑。

代价：

- 后端判题前必须剥离标注标记，避免污染答案。
- 前端展示答案时也要解析并隐藏标记，只显示纯答案内容。

### 备选方案：新增数据库字段保存标注对象名

为练习记录和考试答题记录新增 `image_annotations` 字段，DTO 和 VO 均增加该字段。

优点是答案和附件彻底分离；缺点是需要数据库迁移、实体改造、接口改造，范围明显更大。当前项目已有手写作文画布通过答案标记保存对象名的先例，因此本次不优先采用。

### 不采用方案：只保存在前端本地

只保存到浏览器缓存或本地草稿无法满足“提交后查看记录还能看到”，因此不采用。

## 前端设计

### 图片展示组件

改造现有 `QuestionImageList`，或抽出新的 `QuestionImageAnnotator` 组件，负责统一处理题图展示和标注。

组件职责：

- 将 `imageUrls` 中的 MinIO 对象名转换为可预览 URL。
- 控制题图尺寸：桌面端单图最大宽度约 560 到 640 像素，最大高度约 360 到 420 像素；小屏宽度自适应。
- 避免大面积空白容器，图片区域按实际图片比例收缩。
- 在可编辑模式下叠加透明 canvas。
- 在只读模式下叠加已保存的标注层。

建议交互：

- 默认显示图片和工具栏。
- 工具栏提供“画线/浏览”切换、“撤销”、“清空”。
- 第一版重点支持自由画笔和直线辅助线，直线作为数学题默认工具。
- 清空只清空当前图片的标注层，不影响原图和文字答案。

### 作答页接入

练习作答页和考试作答页都维护每道题的图片标注草稿：

```text
imageAnnotationDrafts[questionId][imageObjectName] = transparentPngDataUrl
```

在保存草稿或提交前：

1. 找出当前题目存在标注的图片。
2. 将透明 PNG data URL 转为 File/Blob。
3. 调用现有 `fileApi.upload(file)` 上传到 MinIO。
4. 获取 `annotationObjectName`。
5. 将标注对象名写入答案 payload 的结构化标记。

没有作图痕迹时，不上传标注层，也不写入标记。

### 答案标记工具

新增通用工具函数，用于构建、解析和剥离图片标注标记：

- `buildImageAnnotationPayload(text, annotations)`
- `parseImageAnnotationPayload(value)`
- `stripImageAnnotationPayload(value)`

该工具应和现有作文手写画布标记兼容，保证同一份 `userAnswer` 中可以同时存在文本答案、作文手写画布标记、图片标注标记。

## 后端设计

### 判题剥离

后端自动判题前，统一从 `userAnswer` 中剥离图片标注标记，只把纯答案交给 `AnswerJudgeSupport`。

适用位置：

- 练习提交判题。
- 练习草稿转记录时如果涉及判题，也使用纯答案。
- 考试提交判题。

保存到记录表时仍保存完整 `userAnswer`，包含图片标注标记，供记录页回看。

### 记录详情返回题图

当前作答接口已经返回题目图片，但记录详情接口还需要补充题图信息：

- 练习记录 VO 增加 `imageUrls` 和 `imageDesc`。
- 考试记录题目 VO 增加 `imageUrls` 和 `imageDesc`。
- 组装记录详情时从 `Question` 实体写入这两个字段。

这样记录页可以同时拿到原图对象名和答案中的标注层对象名。

## 记录回看设计

练习记录详情和考试记录详情展示时：

1. 使用记录接口返回的 `imageUrls` 渲染原题图片。
2. 解析 `userAnswer` 中的图片标注标记。
3. 按 `imageObjectName` 匹配对应的 `annotationObjectName`。
4. 下载或预览标注层图片。
5. 以只读模式叠加展示原图和标注层。
6. 展示答案文本时使用剥离标记后的纯答案，避免把结构化标记暴露给用户。

旧记录没有 `imageUrls` 或没有标注标记时，页面保持现有展示逻辑。

## 错误处理

- 标注层上传失败时，阻止保存/提交并提示用户“图片辅助线保存失败，请稍后重试”。
- 单张图标注失败时，不提交半成品答案，避免记录中缺失用户认为已保存的辅助线。
- 标注对象名解析失败时，记录页仍展示原题图片和纯答案，并隐藏异常标记。
- 预览 URL 获取失败时，展示图片加载失败状态，不影响文字答案查看。

## 测试计划

前端：

- 测试图片标注标记的构建、解析、剥离。
- 测试没有标注时不生成标记。
- 测试多个题图的标注对象名可以按原图对象名匹配。
- 测试练习和考试提交前会上传标注层并写入答案 payload。
- 运行 `npm run lint` 和 `npm run build`。

后端：

- 测试判题前会剥离图片标注标记。
- 测试保存记录时仍保留完整 `userAnswer`。
- 测试练习记录详情返回 `imageUrls` 和 `imageDesc`。
- 测试考试记录详情返回 `imageUrls` 和 `imageDesc`。

人工验收：

- 作答页题图尺寸紧凑，没有截图中那种大面积空白。
- 在题图上画辅助线后保存练习草稿，刷新后仍可看到。
- 提交练习后查看练习记录，能看到原图和辅助线。
- 提交考试后查看考试记录，能看到原图和辅助线。
- 自动判题题目不会因为答案中包含标注标记而误判。
